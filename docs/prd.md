# 产品需求文档 (PRD) —— 「宠爱智囊 (PetAI Mind)」智能健康管理系统

## 1. 文档基本信息与产品定位

### 1.1 文档信息

| 字段 | 内容 |
|------|------|
| 产品名称 | 宠爱智囊 (PetAI Mind) |
| 版本号 | V1.0 (MVP) |
| 产品形态 | Web 端响应式网页 (Responsive Web Application) |
| 目标用户 | 追求科学养宠、伴有日常养育焦虑、需要快速解读健康指标的宠物主 |

| 产品负责人 | Gzy |

### 1.2 产品核心价值

在当前宠物红海市场中，泛滥的"宠物人格化聊天"往往流于娱乐化，缺乏长期刚需粘性。本产品切入**高频、刚需且能体现 AI 多模态核心能力**的医疗健康与行为管理场景。

**一句话定位**：一站式智能宠物健康与行为管理看板 —— 用 AI 让每位宠物主都拥有"随身兽医顾问"。

### 1.3 竞品差异化分析

| 维度 | 传统宠物App (波奇/有宠) | 在线问诊平台 (小荷宠医/蘑菇宠医) | AI宠物工具 (PetAI类) | **宠爱智囊** |
|------|------------------------|--------------------------------|---------------------|-------------|
| 核心功能 | 社交/电商/资讯 | 真人兽医图文问诊 | 单一AI聊天/识图 | AI多模态问诊 + 数据看板 + 社区 + 商城 |
| 响应速度 | - | 排队等待15-30分钟 | 即时但不专业 | AI即时响应，首字<2秒，结合档案个性化 |
| 成本 | 免费/广告 | 单次30-100元 | 免费/订阅 | 免费基础 + 积分兑换 + 增值订阅 |
| 数据沉淀 | 无 | 无结构化档案 | 无 | 完整健康档案 + 趋势分析 + 对话历史 |
| 社区生态 | 有但泛化 | 无 | 无 | 垂直宠物健康问答 + 积分激励 |
| 商业闭环 | 电商导流 | 问诊收费 | 无 | AI诊断→商品推荐→CPS佣金 |
| 可用性 | 7x24 | 工作时间 | 7x24 | 7x24 |

**核心差异化卖点**：
1. **Web大屏数据看板**：体重趋势、健康指标可视化，竞品均为移动端小屏
2. **多Agent协作架构**：不同症状路由到专业Agent，非单一通用模型
3. **档案驱动的个性化**：AI诊断结合宠物品种、年龄、病史，非千篇一律
4. **社区+积分+商城闭环**：问答赚积分、积分兑商品、诊断推商品，形成完整生态

## 2. 目标用户画像与用户旅程

### 2.1 用户画像

**主要用户：焦虑型新手宠物主（占比约60%）**
- 年龄：22-35岁，一二线城市
- 特征：第一次养宠，遇到问题习惯先搜索/问AI
- 痛点：看不懂化验单、不确定是否需要就医、不知道该喂多少
- 场景：猫咪呕吐了→拍照上传→快速判断是否紧急

**次要用户：经验型宠物主（占比约30%）**
- 年龄：28-45岁
- 特征：有多年养宠经验，追求科学喂养
- 痛点：想要系统化的健康档案管理、精准的营养配比
- 场景：定期录入体重→查看趋势→调整食谱

**边缘用户：宠物行业从业者（占比约10%）**
- 宠物店主、宠物博主
- 需求：批量管理多只宠物、生成专业内容

### 2.2 用户故事

**US-01：新用户首次建档**
> 作为一位刚领养猫咪的新手主人，我想快速录入猫咪的基本信息，以便系统能为我提供个性化的健康建议。
> - 验收标准：3分钟内完成建档，品种选择覆盖常见100+品种

**US-02：症状图片问诊**
> 作为一位发现猫咪皮肤异常的主人，我想拍照上传并描述症状，以便快速了解是否需要立即就医。
> - 验收标准：上传图片后30秒内获得结构化诊断报告，含紧急程度标签

**US-03：化验单解读**
> 作为一位拿到猫咪体检报告但看不懂的主人，我想上传化验单照片，以便用大白话理解每项指标的含义。
> - 验收标准：OCR识别率>95%，输出含指标释义和综合建议

**US-04：体重趋势监控**
> 作为一位关注猫咪体重的主人，我想定期录入体重并查看趋势图，以便及时发现肥胖或消瘦风险。
> - 验收标准：图表支持按周/月/年查看，自动对比同品种标准曲线

**US-05：智能食谱生成**
> 作为一位想给猫咪做自制辅食的主人，我想根据猫咪体重和状态获得精准的食物配比建议。
> - 验收标准：输出含具体克数和食材清单，热量计算误差<5%

**US-06：AI对话式问诊**
> 作为一位发现猫咪行为异常但不确定原因的主人，我想通过多轮对话让AI帮我逐步分析问题。
> - 验收标准：支持文字+图片多轮对话，2-4轮追问后给出结构化诊断报告

**US-07：社区求助**
> 作为一位遇到养宠难题的主人，我想在社区发帖求助，获得其他有经验宠物主的建议。
> - 验收标准：发帖后24小时内至少获得1个回答，回答可被采纳

**US-08：积分兑换**
> 作为一位积极回答问题的用户，我想用赚取的积分兑换宠物零食和玩具。
> - 验收标准：积分兑换流程顺畅，商品7天内发货

**US-09：商品推荐购买**
> 作为一位刚获得AI诊断报告的主人，我想直接购买AI推荐的相关商品。
> - 验收标准：推荐商品与诊断结果相关，点击跳转第三方平台可正常购买

### 3.3 关键技术方案

#### 3.3.1 SSE 流式传输方案
```
前端: EventSource / fetch + ReadableStream
后端: FastAPI StreamingResponse + asyncio

断线重连策略:
- 前端自动重连，间隔指数退避 (1s, 2s, 4s, 最大30s)
- 消息带唯一ID，重连时通过Last-Event-ID恢复
- 超过3次重连失败，提示用户刷新
```

#### 3.3.2 多模态图片处理流程
```
用户上传图片 → 前端压缩(webp, max 1MB) → 后端接收
    → 存储至OSS(预签名URL) → 返回图片URL
    → 调用多模态大模型(传入图片URL + 宠物档案上下文)
    → SSE流式返回结构化诊断结果
```

#### 3.3.3 RAG 知识库构建
```
数据源:
- 公开兽医教材/指南 (需确认版权)
- 宠物营养学标准 (NRC/AAFCO)
- 常见疾病知识图谱

处理流程:
文档分块 → Embedding(OpenAI/Claude) → 存入ChromaDB/PGVector
查询: 用户问题 Embedding → 相似度Top-K → 作为上下文注入Prompt
```

### 4.2 关键交互流程

**传图识病交互流程**：
```
1. 用户进入"AI问诊大厅" → 选择"传图识病"
2. 填写症状描述（必填，至少10字）→ 上传图片（选填，1-3张）
3. 点击"开始诊断" → 按钮变为loading状态，显示"AI正在分析..."
4. SSE流式输出 → 打字机效果逐步展示：
   ├── [1-3s] "正在读取宠物档案..."
   ├── [3-8s] "正在分析图片特征..."
   └── [8-20s] 逐段输出诊断报告（疑似病因→紧急程度→护理建议→免责）
5. 报告完成 → 底部出现"保存报告"和"分享"按钮
6. 异常情况：
   ├── 图片上传失败 → 提示重试，保留已填文本
   ├── AI响应超时(>60s) → 提示"分析较慢，请稍候"，保留连接
   └── AI输出异常 → 显示"暂时无法分析，请稍后重试"
```

**化验单解读交互流程**：
```
1. 用户进入"AI问诊大厅" → 选择"化验单解读"
2. 上传化验单图片（支持拍照/相册，单张）
3. 系统自动OCR识别 → 显示"正在识别化验单..."
4. 识别完成 → 展示结构化表格：
   ├── 指标名称 | 测定值 | 参考范围 | 状态(↑正常↓) | 白话释义
   └── 异常指标高亮标红
5. 表格下方 → AI综合分析：饮食建议 + 复查提醒
6. 异常情况：
   ├── 图片模糊/无法识别 → 提示"请上传更清晰的照片"
   ├── 非化验单图片 → 提示"未检测到化验单内容"
   └── 部分指标识别失败 → 展示已识别部分，标注"以下指标未能识别"
```

## 5. 核心功能需求

### 5.1 模块一：宠物动态智能档案管理

**功能目标**：收集、清洗并结构化存储宠物背景数据，作为 AI Agent 精准诊断的"上下文底座"。

#### 5.1.1 宠物基础建档数据项

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 宠物昵称 | 文本 | ✅ | 最长20字符 |
| 宠物品种 | 二级联动下拉 | ✅ | 猫→布偶猫/英短等；狗→金毛/柴犬等，覆盖100+常见品种 |
| 宠物性别 | 单选 | ✅ | 雄性 / 雌性 |
| 是否绝育 | 单选 | ✅ | 已绝育 / 未绝育 |
| 生日/年龄 | 日期选择器 | ✅ | 自动转化为周岁/月龄 |
| 体重 | 数值(kg) | ✅ | 最近一次体重，支持手动更新 |
| 历史病史 | 多选标签+文本 | 选填 | 预设常见病标签 + 自由文本 |
| 过敏史 | 多选标签+文本 | 选填 | 预设常见过敏原 + 自由文本 |

### 5.3 模块三：自适应智能食谱与能量配置器

**功能目标**：将兽医营养学公式代码化，通过 AI 生成兼顾精准热量与口味偏好的动态鲜食谱。

#### 5.3.1 核心算法逻辑

**Step 1：计算基础能量需求 (RER)**
$$RER = 70 \times (W)^{0.75}$$
其中 W 为宠物最新体重（kg）

**Step 2：获取生命阶段系数 (Factor)**

| 生命阶段 | 系数范围 | MVP默认值 |
|---------|---------|----------|
| 幼犬/幼猫（<1岁） | 2.0 - 3.0 | 2.5 |
| 成年已绝育 | 1.2 - 1.4 | 1.3 |
| 成年未绝育 | 1.4 - 1.6 | 1.5 |
| 减重期 | 0.8 - 1.0 | 1.0 |
| 老年（>10岁） | 1.0 - 1.2 | 1.1 |

**Step 3：计算维持能量需求 (MER)**
$$MER = RER \times Factor$$

**Step 4：AI食谱生成**
- 根据MER计算结果，结合宠物品种偏好和过敏史
- 输出具体食物种类和克数
- 标注营养素占比（蛋白质/脂肪/碳水）

#### 5.5.5 边界条件与错误处理

| 场景 | 处理方式 |
|------|---------|
| 商品链接失效 | 定期检测，失效商品自动下架，替换为同类商品 |
| 用户投诉商品质量 | 引导联系第三方平台售后，平台不承担商品责任 |
| 推荐不相关 | 用户可点击"不感兴趣"，优化推荐算法 |

| 第三方平台接口异常 | 降级为静态推荐列表，不影响核心功能 |

**功能目标**：将兽医营养学公式代码化，通过 AI 生成兼顾精准热量与口味偏好的动态鲜食谱。

#### 5.3.1 核心算法逻辑

**Step 1：计算基础能量需求 (RER)**
$$RER = 70 \times (W)^{0.75}$$
其中 W 为宠物最新体重（kg）

**Step 2：获取生命阶段系数 (Factor)**

| 生命阶段 | 系数范围 | MVP默认值 |
|---------|---------|----------|
| 幼犬/幼猫（<1岁） | 2.0 - 3.0 | 2.5 |
| 成年已绝育 | 1.2 - 1.4 | 1.3 |
| 成年未绝育 | 1.4 - 1.6 | 1.5 |
| 减重期 | 0.8 - 1.0 | 1.0 |
| 老年（>10岁） | 1.0 - 1.2 | 1.1 |

**Step 3：计算维持能量需求 (MER)**
$$MER = RER \times Factor$$

**Step 4：AI食谱生成**
- 根据MER计算结果，结合宠物品种偏好和过敏史
- 输出具体食物种类和克数
- 标注营养素占比（蛋白质/脂肪/碳水）
